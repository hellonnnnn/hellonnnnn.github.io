2. 惰性分配（Lazy Allocation）后的页表状态

（1）初始状态（调用 sbrk 扩展堆）

  • 虚拟地址已经增加（p->sz 增大）

  • 物理页尚未分配

  • 页表未映射 PTE_V=0，walk/pte 返回 0

  • 特点：

    • 进程可以“申请”任意大小的 heap，但占用物理内存为 0
    • 节省内存

（2）第一次访问未分配页（page fault）

  • 用户访问未分配的虚拟地址 → CPU 触发 page fault（scause=13/15）

  • usertrap() 检测到地址合法（faulting_addr < p->sz）

  • 调用 kalloc() 分配物理页 → mappages 映射到页表

  • 页表 PTE 更新：

    • PTE_V = 1
    • PTE_R | PTE_W | PTE_U 权限

  • 特点：

    • 只分配访问过的页
    • 页表中未访问页仍为 0

（3）系统调用访问 lazy 页

  • copyin/copyout/copyinstr 访问地址，如果物理页未分配
  • 调用 lazy_alloc() 分配物理页
  • 页表更新同上

（4）释放内存（uvmunmap / uvmdealloc）

  • 遍历页表

  • 已分配页：释放物理页，PTE=0

  • 未分配页（PTE_V=0）：跳过

  • 特点：

    • 避免 panic
    • 支持 shrink 堆或释放进程

3. 写时复制（COW）下的页表状态

（1）fork 时

  • 调用 uvmcopy() 复制页表

  • COW 原理：

    • 对于已分配页（PTE_V=1）：

      • 标记为只读（去掉 PTE_W）
      • 设置共享物理页

    • 对于未分配页（PTE_V=0）：

      • 不分配物理页
      • 子进程页表同样为 0

（2）子进程第一次写

  • 写访问只读页 → CPU 触发 page fault（scause=15）

  • 内核处理：

    • 调用 kalloc() 分配新物理页

    • 拷贝原物理页内容到新页（memmove）

    • 更新 PTE：

      • PTE_W = 1
      • PTE_U = 1
      • PTE_V = 1

  • 原进程页表仍指向旧物理页

  • 特点：

    • 避免 fork 时复制整个 heap
    • 只在写时才复制页，提高效率

4. 页表使用状态总结表

页表类型 / 状态 PTE_V PTE_W PTE_R PTE_U 对应物理页 说明
未访问 lazy 页 0 0 0 0 无 仅虚拟地址存在，物理页未分配
已访问 lazy 页 1 1 1 1 分配 用户首次访问后分配物理页
COW 页（fork 后） 1 0 1 1 共享 子进程和父进程只读共享物理页
COW 写后页 1 1 1 1 新分配 写入后，子进程拥有独立物理页
已释放页 0 0 0 0 已释放 heap shrink 或进程退出

5. 流程示意

  1. sbrk() / heap grow → 虚拟地址增加 → 页表 PTE_V=0
  2. 用户访问 lazy 页 → page fault → kalloc() → mappages() → PTE_V=1
  3. fork() → 复制页表 → 已分配页标记只读 → 未分配页保持 0
  4. 子进程写 COW 页 → page fault → 分配新物理页 → 更新 PTE_W=1

6. 总结

  • 惰性分配：延迟物理页分配，节省内存，PTE_V=0 表示未分配。
  • 写时复制：fork 时不立即复制物理页，通过 PTE_W=0 + page fault 实现按需复制。
  • 页表中 PTE_V=0 的页 表示虚拟地址存在但未分配物理页。
  • 系统调用访问这些页必须支持 lazy allocation。
  • 释放页 时要区分已分配和未分配页，避免 panic。
