磁盘层和cache层

2026年02月14日 703 字 大概 3 分钟

我用最清晰、最贴合 xv6 源码的方式，给你一次性讲透 磁盘层（disk layer） 和 缓冲区缓存层（buffer cache layer），你可以直接用来理解 xv6 文件系统整体架构。

一、磁盘层（disk layer）

作用：最底层，只负责和硬件磁盘交互，只做「读一个块 / 写一个块」。

1. 定位

  • 在 xv6 中：ide.c（IDE 硬盘驱动）
  • 是文件系统最底层，上层（buffer cache、log、fs）都不直接碰磁盘。

2. 功能只有两个

  1. 读磁盘块：从磁盘第 blockno 号块读到内存。
  2. 写磁盘块：把内存里的一块数据写到磁盘第 blockno 号块。

3. 特点

  • 只认 512 字节一个块（xv6 磁盘块大小）。
  • 不理解文件、inode、目录、日志——它只管块号和数据。
  • 是同步/阻塞的：发起读写，必须等磁盘完成。
  • 不缓存、不合并、不优化。

4. 对外接口（简化）

  • write=0：读磁盘
  • write=1：写磁盘

二、缓存层（buffer cache layer）

作用：给磁盘加一层内存缓存，统一管理磁盘块，防止重复 I/O，保证互斥。

对应文件：bio.c

1. 为什么需要 cache？

  1. 磁盘太慢，内存快 → 缓存热块
  2. 多个进程同时写同一块 → 必须互斥访问
  3. 文件系统需要拿到一块“可修改的内存缓冲区”来操作 inode、bitmap 等

2. 核心结构：buffer（buf）

每个 buf 代表：

  • 磁盘上的一个块
  • 内存中的一份副本
  • 状态、锁、引用计数、链表指针

3. 核心两个接口（最重要）

1）bread() —— 读块

工作流程：

  1. 去 cache 里找这个块号是否已经在内存
  2. 命中：直接返回
  3. 不命中：
    • 找一个空闲 buf（LRU 淘汰）
    • 调用 ide_rw 从磁盘读入
    • 标记 B_VALID
    • 返回 buf

2）bwrite() —— 写块

  • 把内存中的 buf 数据写到磁盘
  • 只写真正被修改过的（B_DIRTY）

3）brelse()

  • 释放缓冲区（解锁）
  • 放回到 LRU 链表

4. 缓存层三大作用

  1. 减少磁盘 I/O
    多次读同一块，只访问一次磁盘。

  2. 提供互斥访问
    每个 buf 有锁，同一时间只有一个进程能修改同一块。

  3. 提供统一的内存缓冲区
    上层（日志、inode、文件）只操作 buf->data，不关心磁盘。

三、磁盘层 ↔ 缓存层 的关系（超级清晰）

一句话总结：

  • 磁盘层：只做原始块读写。
  • 缓存层：管理磁盘块，提供缓存、互斥、内存副本。
