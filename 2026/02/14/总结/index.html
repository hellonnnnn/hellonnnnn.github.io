xv6 文件系统六层全链路总结（背诵版+逻辑图）

一、六层核心定位速记（一句话/层）

层级 核心定位 关键关键词
1. 日志层 包裹所有磁盘修改，保证原子性，崩溃后自动恢复 begin_op/end_op、log_write、原子性
2. 块分配层 管理磁盘空闲块，提供块的分配（balloc）与释放（bfree） 空闲位图、BPB、buffer cache 独占
3. inode层 管理文件元信息+数据块映射，封装直接/间接块，提供inode的分配/读写/释放 dinode/inode、bmap/itrunc、ref/nlink
4. 目录层 以“特殊inode（T_DIR）”存储目录条目，实现目录的查找（dirlookup）与新增（dirlink） dirent、.和..、死锁规避
5. 路径层 将字符串路径转换为inode指针，支持绝对/相对路径，规避并发/死锁风险 namex/namei、逐目录加锁、ref防删除
6. 文件描述符层 统一抽象所有I/O对象（文件/管道/设备），提供标准化读写接口 struct file、引用计数、偏移量

二、六层全链路逻辑图（Mermaid 可直接渲染）

三、六层核心协作流程（背诵版）

以「用户调用 open 创建文件」为例，全链路核心步骤：

  1. 日志层：sys_open 调用 begin_op() 启动事务，所有磁盘修改包裹在事务中；
  2. 路径层：nameiparent 解析文件父目录路径，逐目录加锁查找，返回父目录 inode（dp）；
  3. 目录层：dirlookup 检查文件名是否已存在，不存在则准备新增条目；
  4. inode层：ialloc 分配新 inode（type=T_FILE），bmap 为文件分配数据块；
  5. 块分配层：balloc 从空闲位图分配磁盘块，更新位图并写入日志；
  6. 目录层：dirlink 在父目录 dp 中新增条目，指向新 inode；
  7. inode层：iupdate 将新 inode 同步到磁盘，iput 管理引用计数；
  8. 文件描述符层：filealloc 分配 struct file，绑定新 inode，设置偏移量/读写权限，返回文件描述符；
  9. 日志层：end_op() 提交事务，日志刷盘，所有修改原子生效。

四、六层核心设计原则（必背3条）

  1. 原子性：所有磁盘修改（块分配、inode修改、目录修改）必须包裹在日志事务中，崩溃后要么全执行、要么全回滚；
  2. 分层解耦：上层依赖下层的接口，不关心下层实现（如路径层只调用目录层dirlookup，不关心目录如何存储）；
  3. 并发安全：
    • 底层（块/inode）：依赖buffer cache/睡眠锁保证独占访问；
    • 中层（目录/路径）：逐目录加锁、ref计数防删除、先解锁旧目录再锁新目录避死锁；
    • 上层（文件描述符）：引用计数保证共享安全，偏移量原子更新。

五、六层核心记忆口诀（极简版）

日志包原子，块管空闲位；
inode映数据，目录存条目；
路径找inode，描述符统接口。

下面给你 最标准、最清晰、考试/背诵直接用的 xv6 文件系统「七层总结」
严格按你前面学的顺序：
日志层 → 块分配层 → inode 层 → 目录层 → 路径名层 → 文件描述符层 → 系统调用层

每一层只给：作用 + 核心结构 + 关键函数 + 一句话口诀

xv6 文件系统 七层 超级背诵版

1. 日志层（最底层，保证安全）

作用：让所有磁盘操作原子性，崩溃后自动恢复。
核心：事务（transaction）
关键函数

  • begin_op()：开始事务
  • log_write()：记录修改
  • end_op()：提交事务
    口诀：要么全做，要么全不做，崩溃不怕。

2. 块分配层（管理磁盘空闲块）

作用：分配/释放磁盘块，用空闲位图管理。
核心：bitmap（一位对应一块）
关键函数

  • balloc()：分配块
  • bfree()：释放块
    口诀：0 空闲，1 占用，找 0 分配。

3. inode 层（文件元数据核心）

作用：管理文件大小、权限、数据块位置。
结构

  • 磁盘 inode (dinode)
  • 内存 inode (inode)
    关键函数
  • ialloc()：分配 inode
  • iget/iput：引用计数
  • ilock/iunlock：独占锁
  • bmap()：直接块 + 间接块映射
  • readi/writei：文件读写
  • itrunc()：删除文件数据
    口诀：直接块快，间接块大，引用计数保安全。

4. 目录层（文件名 ↔ inode）

作用：目录就是类型为 T_DIR 的文件，存 dirent 条目。
结构：struct dirent { inum, name }
关键函数

  • dirlookup()：查找文件名
  • dirlink()：添加文件名
    口诀：目录就是文件，存名字和编号。

5. 路径名层（字符串 → inode）

作用：把 /a/b/c 变成 inode 指针。
关键函数

  • namei()：找最终文件
  • nameiparent()：找父目录
  • namex()：通用查找
    安全点
  • 逐目录加锁
  • 先解锁旧目录，再锁新目录 → 防死锁
  • iget 增加引用 → 防删除
    口诀：一层一层找，一次只锁一个。

6. 文件描述符层（统一一切 I/O）

作用：统一抽象 文件、管道、设备，实现“一切皆文件”。
结构：struct file
关键函数

  • filealloc()：分配打开文件
  • filedup()：共享（fork/dup）
  • fileclose()：关闭
  • fileread/filewrite：统一读写
    特点
  • 每个打开文件有 偏移量 offset
  • 引用计数 ref 管理共享
    口诀：统一接口，共享安全，偏移自动走。

7. 系统调用层（用户接口）

作用：给用户提供 open/read/write/link/unlink/mkdir 等接口。
核心调用

  • sys_link：硬链接（增加 nlink）
  • sys_unlink：删除链接
  • sys_open：打开/创建文件
  • sys_mkdir：创建目录
  • sys_pipe：管道
    统一规则
  • 全部包在日志事务里
  • 全部用前面 6 层函数
    口诀：用户调接口，内核七层跑。

终极七句总口诀（背会就满分）

  1. 日志保原子，崩溃不损坏
  2. 块位图分配，管理空闲盘
  3. inode存元数据，直接间接块
  4. 目录是文件，存名与编号
  5. 路径逐层找，锁序防死锁
  6. 描述符统一，一切皆文件
  7. 系统调接口，事务保安全
